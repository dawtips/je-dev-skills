# Emitted Artifact Patterns

Use these patterns when reviewing or editing files generated by
`agent-build-scaffold`.

## Subagent: `.claude/agents/<id>.md`

```markdown
---
name: <slug>
description: Handles the <slug> step for the scaffolded workflow
model: <tier-or-runtime-name>
tools: Read, Bash
---

# <id>

## Objective
The single job this subagent owns.

## Output Format
The required shape, often JSON.

## Boundaries
What this subagent must not do.

## Notes
- Recommended effort: <effort>.
- Keep this subagent one level deep.
```

The required four-part contract is objective, output format, tools, and
boundaries. `tools` is frontmatter because Claude Code uses it for permissioning;
the other three fields stay in the body because they instruct behavior.

## Deterministic Step Script: `.claude/scripts/<id>.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

# AUTO-GENERATED placeholder by agent-build-scaffold. Fill in the real command below.
# Step: <id>
# Rationale: <why deterministic>
echo 'TODO: implement this deterministic step'
```

Side-effecting scripts add an idempotency marker:

```bash
IDEMPOTENCY_KEY="${ORDER_ID:?set ORDER_ID}"
SAFE_IDEMPOTENCY_KEY="$(printf '%s' "$IDEMPOTENCY_KEY" | sed 's/[^A-Za-z0-9_.-]/_/g')"
MARKER=".agent-build-state/<step-id>-${SAFE_IDEMPOTENCY_KEY}.done"
mkdir -p .agent-build-state
if [ -f "$MARKER" ]; then
  exit 0
fi

echo 'TODO: implement the side-effecting command'
touch "$MARKER"
```

## Rubric Gate Script

Rubric gate scripts are shell scripts under `.claude/hooks/`. They are named by
the generated command and invoked explicitly after
`.agent-build-state/<rubric>.score` has been written.

- Exit `0`: pass.
- Exit `2`: block.
- Score input: `.agent-build-state/<rubric>.score`.
- Project root: reads `CLAUDE_PROJECT_DIR` when present, otherwise current
  working directory.

The scaffolder does not auto-register project hooks. Current Claude Code project
hooks are session/project-level lifecycle events; auto-wiring a score-file gate
to `SubagentStop` can block before the parent workflow writes the score.

## Entry-Point Command

The generated `.claude/commands/<workflow>.md` command is an orchestration plan,
not a hidden executor. It lists steps in blueprint order:

- Agentic step: dispatch the paired subagent.
- Deterministic step: run the generated script.
- Termination: stop when step termination or gates fail.
- Gates: score-file names, thresholds, and gate scripts to run.
- Side effects: idempotency key, retry policy, and rollback note.
- Nesting: one level deep only.

## State Directory

`.agent-build-state/` stores idempotency markers and rubric score files. Generated
skills instruct users to add it to `.gitignore` because it is runtime state, not
source.
